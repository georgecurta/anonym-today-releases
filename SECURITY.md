# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in anonym.today Desktop, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security concerns to: **security@anonym.today**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes

We will acknowledge receipt within 48 hours and provide updates on our investigation.

## Security Features

- **End-to-end encryption** for document processing
- **No server-side storage** of user documents
- **BIP39 secure vault** with 24-word recovery phrase
- **Local-first architecture** - sensitive operations happen on your device

## Responsible Disclosure

We appreciate security researchers who help keep anonym.today safe. We commit to:

- Responding promptly to vulnerability reports
- Keeping reporters informed of remediation progress
- Crediting researchers (with permission) in release notes
